React

What is React ?

  • a fast JavaScript library for creating user interfaces
  • created by Facebook to facilitate the creation of interactive, stateful and reusable UI components
  • can be rendered on both client side and server side

Key concepts

  • Virtual DOM - selectively renders only the parts of DOM that have changed
  • Components - UI elements are constructed with React components
  • Props - the attributes passed down to the component
  • Unidirectional data flow - in a React app, data flows one way from parent to children components
  • JSX - JavaScript XML - lets you write HTML-ish code in your JavaScript
  • State - the current state of a component

Example

What's there to like ?

  • Though it was a little more difficult to start off, once you get the concepts and how it works, it's really a great and easy to use tool
  • It brings structure and order to the project
  • The idea of components is a plus, that really makes the application fast

Do what I say, don't do what I do

  • The store ? We didn't think it through
  • Didn't consistently use PropTypes for all of our components
  • Didn't apply the dumb/stateless component - container component architecture thoroughly
  • We started head first into programming, without at least going through the major components of the app
  • onEvent={this.eventHandler.bind(this)} in the components
  • In some cases we placed component state directly into the store

Conclusion

  • Model your store from the start, at least the major structures. The rest will easily fall into place.
  • Break your app into components from the very beginning. Not all the visual components, but enough to get an idea about the appropriate structure you could use.
  • PropTypes, PropTypes, PropTypes
  • You should really start using Immutable.js.
